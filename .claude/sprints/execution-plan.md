# ReHome MVP Execution Plan

## Goal
Ship a demo-ready donor + NGO loop in 3 days with working Supabase persistence and role-based flows.

## Day 1 Foundation
- Task 1.1: Repo audit + cut list
- Task 1.2: MVP schema migration + seed
- Task 1.3: Auth role routing
- Task 1.4: Scan capture + review draft state
- Task 1.5: On-device AI wiring
- Task 1.6: NGO pick + submit function
- Task 1.7: NGO inbox + accept/decline

## Day 2 Loop Closure
- Pickup scheduling and signed confirmation token
- Pickup confirm page + points award transaction
- Donor donations/history/impact pages
- Rewards redemption (MVP)
- NGO settings/capacity
- Bug bash + staging deploy

## Day 3 Investor Readiness
- Realistic seeded data
- Impact page polish
- Lightweight admin tools
- Performance pass
- Landing/onboarding improvements
- Demo script and recording assets

## Operating Rules
- Prefer shipping end-to-end slices over broad unfinished UI
- If a task overruns by >50 percent, cut and move to next MVP-critical item
- Preserve existing code unless directly required for MVP
- Use feature flags/route guards to hide non-MVP features instead of deleting code
